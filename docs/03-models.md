## 1. Core Data Models

### 1.1 ColumnSchema

Represents the definition of a single column in the table.

```ts
interface ColumnSchema {
  id: string;                                              // Stable unique identifier
  type: "string" | "number" | "boolean" | "date" | "custom";
  title?: string;                                         // Display label
  width?: number;                                         // Default pixel width
  default?: any;                                          // Default cell value (used in lazy resolution)
  meta?: Record<string, any>;                             // Extensible: renderer, validation rules, etc.
}
```

**Invariants:**
- `id` must be unique within a `TableSchema`
- `id` must be a non-empty string
- `type` must be one of the defined literals
- `width`, if defined, must be a positive number

---

### 1.2 TableSchema

Represents the complete structural definition of the table.

```ts
interface TableSchema {
  columns: Record<string, ColumnSchema>;  // Map from column ID → ColumnSchema
  columnOrder: string[];                  // Ordered list of column IDs (defines display order)
  version: number;                        // Monotonic integer; incremented on every SchemaOperation
}
```

**Invariants:**
- Every entry in `columnOrder` must exist as a key in `columns`
- Every key in `columns` must appear in `columnOrder`
- `version` must be >= 0 and must only increase
- `columns` must not contain duplicate IDs

---

### 1.3 Cell

Represents a single data value within a row.

```ts
interface Cell {
  value: any;                    // The actual data value (semantically typed per column)
  meta?: Record<string, any>;   // Optional metadata (e.g., display hints, validation state)
}
```

**Invariants:**
- `value` may be `null` (represents empty/unset cell)
- `meta` is open-ended; consumers must not assume specific keys

---

### 1.4 Row

Represents a single data record in the table.

```ts
interface Row {
  id: string;                          // Stable unique identifier (backend-assigned UUID)
  cells: Record<string, Cell>;         // Map from column ID → Cell
}
```

**Invariants:**
- `id` must be unique within the table
- `cells` keys should correspond to known column IDs, but may be sparse (lazy resolution fills gaps)
- A row with empty `cells` is valid (no backfill required on row creation)

---

### 1.5 TableState

The complete in-memory state of the table at any point in time.

```ts
interface TableState {
  schema: TableSchema;
  rows: Map<string, Row>;    // Map from row ID → Row (O(1) lookup)
  rowOrder: string[];        // Ordered list of row IDs (defines display order)
}
```

**Invariants:**
- Every entry in `rowOrder` must exist as a key in `rows`
- Every key in `rows` must appear in `rowOrder`
- `rowOrder.length === rows.size`

## 2. Operation Models

### 2.1 BaseOperation

All operations share this base shape.

```ts
interface BaseOperation {
  id: string;                                            // Unique operation ID (client-generated, e.g., nanoid)
  type: string;                                          // Discriminator; narrowed by union types
  ts?: number;                                           // Optional logical timestamp (Lamport clock or Date.now())
  source: "client" | "server";                          // Origin of the operation
  status?: "pending" | "applying" | "confirmed" | "error"; // Lifecycle state (managed by OperationManager)
  target?: TargetDescriptor;                             // Scope of the operation (for loading state derivation)
  meta?: Record<string, any>;                            // Extensible; for tracing, correlation, etc.
}
```

**Invariants:**
- `id` must be non-empty, unique per operation
- `source` determines trust level (server ops bypass client-side validation)
- `status` is managed externally by `OperationManager`; engine does not set it

---

### 2.2 TargetDescriptor

Describes the scope affected by an operation. Used for deriving loading/pending state in the UI.

```ts
type TargetDescriptor =
  | { type: "cell";   rowId: string; colId: string }
  | { type: "row";    rowId: string }
  | { type: "column"; colId: string }
  | { type: "range";  rowIds: string[]; colIds: string[] };
```

**Invariants:**
- For `cell`: both `rowId` and `colId` must be non-empty
- For `range`: both arrays must be non-empty
- `range` is used for bulk operations

---

### 2.3 DataOperation

Operations that mutate cell data without changing the schema.

```ts
type DataOperation =
  | SetCellOperation
  | BulkUpdateOperation
  | UpdateColumnOperation;

interface SetCellOperation extends BaseOperation {
  type: "set_cell";
  rowId: string;
  colId: string;
  value: any;
}

interface BulkUpdateOperation extends BaseOperation {
  type: "bulk_update";
  updates: Array<{
    rowId: string;
    colId: string;
    value: any;
  }>;
}

interface UpdateColumnOperation extends BaseOperation {
  type: "update_column";
  colId: string;
  values: Record<string, any>;   // Map from rowId → new value for that column
}
```

**Invariants:**
- `SetCellOperation.rowId` and `colId` must be non-empty
- `BulkUpdateOperation.updates` must be a non-empty array
- `UpdateColumnOperation.values` must be a non-empty object

---

### 2.4 SchemaOperation

Operations that mutate the table structure.

```ts
type SchemaOperation =
  | AddColumnOperation
  | RemoveColumnOperation
  | ReorderColumnsOperation
  | AddRowOperation
  | RemoveRowOperation
  | ReorderRowsOperation;

interface AddColumnOperation extends BaseOperation {
  type: "add_column";
  column: ColumnSchema;
  index?: number;    // Insertion position in columnOrder; appended if omitted
}

interface RemoveColumnOperation extends BaseOperation {
  type: "remove_column";
  columnId: string;
}

interface ReorderColumnsOperation extends BaseOperation {
  type: "reorder_columns";
  columnOrder: string[];   // Complete new column order; must contain all existing column IDs
}

interface AddRowOperation extends BaseOperation {
  type: "add_row";
  row: Row;
  index?: number;   // Insertion position in rowOrder; appended if omitted
}

interface RemoveRowOperation extends BaseOperation {
  type: "remove_row";
  rowId: string;
}

interface ReorderRowsOperation extends BaseOperation {
  type: "reorder_rows";
  rowOrder: string[];   // Complete new row order; must contain all existing row IDs
}
```

**Invariants:**
- Every `SchemaOperation` must increment `schema.version` when applied
- `AddColumnOperation.column.id` must not already exist in the schema
- `RemoveColumnOperation.columnId` must exist in the schema
- `ReorderColumnsOperation.columnOrder` must contain exactly the same IDs as the current schema
- `AddRowOperation.row.id` must not already exist in rows
- `RemoveRowOperation.rowId` must exist in rows
- `ReorderRowsOperation.rowOrder` must contain exactly the same IDs as the current rows

---

### 2.5 Operation (Union)

```ts
type Operation = DataOperation | SchemaOperation;
```

## 3. API / Transport Models

### 3.1 SubmitOperationsRequest

```ts
interface SubmitOperationsRequest {
  ops: Operation[];
}
```

---

### 3.2 SubmitOperationsResponse

```ts
interface SubmitOperationsResponse {
  ack: string[];                            // IDs of confirmed operations
  errors?: OperationError[];                // Rejected operations with reasons
  serverOps?: Operation[];                  // Server-initiated corrections or derived ops
}
```

---

### 3.3 OperationError

```ts
interface OperationError {
  opId: string;
  reason: string;
  code?: string;    // Machine-readable error code (optional)
}
```

---

### 3.4 TableLoadResponse

The initial state returned by `GET /table`.

```ts
interface TableLoadResponse {
  schema: TableSchema;
  rows: Record<string, Row>;    // Wire format uses plain object; Engine loads into Map
  rowOrder: string[];
  version: number;
}
```

## 4. Store / Reactive Models

### 4.1 OperationRecord

The OperationManager's internal representation of a tracked operation.

```ts
interface OperationRecord {
  op: Operation;
  status: "pending" | "applying" | "confirmed" | "error";
  addedAt: number;    // timestamp for timeout/retry logic
  error?: OperationError;
}
```

---

### 4.2 TargetIndex

Internal data structure for O(1) pending state lookup by target.

```ts
// Internal to OperationManager — not exposed in public API
type TargetKey = string;   // Serialized from TargetDescriptor, e.g. "cell:r1:A"

interface TargetIndex {
  pendingByTarget: Map<TargetKey, Set<string>>;   // targetKey → Set<opId>
}
```

---

### 4.3 TableStore (Zustand slice)

```ts
interface TableStore {
  // State
  engine: TableEngineImpl;
  manager: OperationManagerImpl;

  // Actions
  dispatch(op: Operation): void;           // Delegates to manager.apply(op)

  // Selectors (derived)
  getCell(rowId: string, colId: string): Cell;
  getSchema(): TableSchema;
  getRowOrder(): string[];
  isPending(target: TargetDescriptor): boolean;
}
```

## 5. Entity Relationships

```
TableSchema
  └── columns: Record<id, ColumnSchema>
  └── columnOrder: string[]
  └── version: number

TableState
  └── schema: TableSchema
  └── rows: Map<id, Row>
  └── rowOrder: string[]

Row
  └── id: string
  └── cells: Record<colId, Cell>

Cell
  └── value: any
  └── meta?: Record<string, any>

Operation (discriminated union)
  └── BaseOperation (common fields: id, type, ts, source, status, target, meta)
  └── DataOperation variants (set_cell, bulk_update, update_column)
  └── SchemaOperation variants (add_column, remove_column, reorder_columns, add_row, remove_row, reorder_rows)
```

## 6. Validation Rules

| Entity | Field | Rule |
|--------|-------|------|
| `ColumnSchema` | `id` | Non-empty, unique within schema |
| `ColumnSchema` | `type` | Must be one of the defined literals |
| `ColumnSchema` | `width` | If defined, must be > 0 |
| `TableSchema` | `columnOrder` | Must exactly match keys in `columns` |
| `TableSchema` | `version` | Must be >= 0; only increases |
| `Row` | `id` | Non-empty, unique within table |
| `BaseOperation` | `id` | Non-empty string |
| `BaseOperation` | `source` | Either `"client"` or `"server"` |
| `AddColumnOperation` | `column.id` | Must not already exist in schema |
| `RemoveColumnOperation` | `columnId` | Must exist in schema |
| `ReorderColumnsOperation` | `columnOrder` | Must be a permutation of existing column IDs |
| `AddRowOperation` | `row.id` | Must not already exist in rows |
| `RemoveRowOperation` | `rowId` | Must exist in rows |
| `BulkUpdateOperation` | `updates` | Non-empty array |
